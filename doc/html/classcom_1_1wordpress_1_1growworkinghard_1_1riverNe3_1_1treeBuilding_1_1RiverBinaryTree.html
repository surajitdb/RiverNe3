void findRoot ( )
private

Find the first root node in the List

The algorithm to build the tree is based on the following idea: I can identify a sub-tree with a root and two children (doesn't matter if they are null or more than two in the reality). So when parsing the List structure I find a root, I have just found the starting point of a new sub-tree. Then I have to look for the two children:

  • if only two children are identified, the sub-tree is complete and the two children become the roots of the following sub-tree;
  • if more than two children are identified, the sub-tree is completed by the real left child and a fake right child (the ghost node) which is added to the List structure. The real left child and the ghost node become the roots of the following sub-tree;
  • if no children are identified, the sub-tree is completed by two null children.

The root node of the sub-tree is removed from the List structure at the end of each parsing loop.

void ricomputeRightChild ( final Geometry  right)
private

Right child becomes a ghost node.

If at least a ghost node is required to build the binary structure, the first right child identified becomes a ghost node.

To easily identify the ghost node in the following parsing of the List structure, the coordinates of the starting point are set equal to the coordinate of the ending point.
