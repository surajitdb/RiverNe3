RunSimulations ( final HashMap< Key, Component >  tree,

final int  threadsNumber 
)

Constructor.

The ExecutorService is passed to the constructor in order to use only one executor for all the tree process. Quoting [2]

Reusing an existing thread instead of creating a new one amortizes thread creation and teardown costs over multiple requests. As an added bonus, since the worker thread often already exists at the time the request arrives, the latency associated with thread creation does not delay task execution, this improving responsiveness.
